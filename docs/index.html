13. Installation Guide

Prerequisites

  • Python 3.11 or higher
  • pip (Python package manager)
  • Git (optional, for version control)

Local Development Setup

# 1. Navigate to project directory
cd idea_tracker

# 2. Create virtual environment
python -m venv venv

# 3. Activate virtual environment
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 4. Install dependencies
pip install -r requirements.txt

# 5. Run the application
python app.py

# 6. Open browser at http://localhost:5000

Configuration

Variable Description Default
SECRET_KEY Flask secret key for sessions 'dev-secret-key...'
DATABASE_URL Database connection string 'sqlite:///ideas.db'

15. Deployment

The application is configured for deployment on Render.com using the included render.yaml file.

Deployment Steps:

Push your code to a GitHub repository
Create a new Web Service on Render.com
Connect your GitHub repository
Render auto-detects the render.yaml configuration
Deployment completes automatically

render.yaml Configuration:

services:
  - type: web
    name: innovation-idea-tracker
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn app:app
    envVars:
      - key: SECRET_KEY
        generateValue: true
      - key: PYTHON_VERSION
        value: 3.11.0

16. Security Considerations

Implemented Security Measures:

Feature Implementation
Password Hashing Werkzeug's PBKDF2 with salt
CSRF Protection Flask-WTF provides tokens for all forms
Session Management Flask-Login secure sessions
Input Validation WTForms validates all user input
Access Control Role-based (submitter, reviewer, admin)

Production Recommendations:

  • Use PostgreSQL instead of SQLite for production
  • Set a strong, unique SECRET_KEY
  • Enable HTTPS for all connections
  • Configure proper logging and monitoring
  • Implement regular database backups
